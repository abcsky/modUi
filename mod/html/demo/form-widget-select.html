使用 HTML模板编写 不用再执行JS,填上适当的参数即可自动生成。其中 mod-xxx 为 js方法中的参数，根据 "-"来区分单词
如 titleLeft 在html中为 mod-title-left 属性。

城市选择

HTML模板写法

<tr>
	<th>
		<label id="city">城市选择</label>
	</th>
	<td>
		<select id="city" name="city" mod-format="city"><option value="广州市">广州市</option></select>
	</td>
</tr>

Javascript写法

//javascript
mod.widget(target,{format:"select"});

mod-widget(target,op) 参数说明

属性 类型 说明
target {object|string} 需要执行模块化的对象
op {object} 参数设置
属性 类型 支持版本 说明
format {string} >= 1.0 格式化类型
= select
城市选择 city
一般选择 不填
required {boolean} >= 1.0 是否为必填项 默认 false
errorText {string} >= 1.0 设置校验内容时错误的提示语
defaultText {string} 设置控件默认的提示语
autocomplete {string} >= 1.0 自动完成（未做功能）
action {string} >= 1.4.8 异步获取数据地址 支持 {#id名称} 方式的模板， 引擎会自动获取 对应元素的 value 值
method {string} >= 1.0 异步获取的方式 GET|POST
actionKey {string} >= 1.0 异步获取时使用的 参数名 如 "username"
actionType {string} >= 1.0 异步获取的途径 ajax|jsonp
linked {string} >= 1.0 关联select（多级选择用函数）
messageKey {string} >= 1.0 错误信息对应的json 属性 默认与Config一致
statusKey {string|number} >= 1.0 状态信息对应的json 属性 默认与Config一致
defaultValue {string} >= 1.37 设置控件默认选中的值
actionCallback {function} >= 1.37 异步获取数据后回调方法
@param json {object} 服务器返回的数据
@param param {object} 提交表单时提交的数据
onresponse(json) {function} >= 1.4.3 提交表单返回后的预处理函数
@param data {object} 服务器返回的数据
@return data {object} 处理后的数据
successCode {string|number} >= 1.4.3 成功返回状态码属性 successCode， 默认值为 bs_config.js 中的设定值 config.ajax.successCode
